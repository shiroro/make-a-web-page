Important Concepts

Lesson 1: The World Wide Web

The World Wide Web

The World Wide Web is a collection of HTML documents started in the early 1990s that now consists of 30+ billion pages. Pages are connected via links, which is how users navigate from one document to another.

HTTP is short for Hypertext Transfer Protocol. It allows users to make requests from the internet, which retrieves files from servers, which are other computers that host files such as web pages.

HTML

HTML is short for Hypertext Markup Language and is the basis of all pages on the World Wide Web. It can consist of text content (what's visible on the page), markup (the code that a computer renders as a web page), references to other types of documents such as images or videos, and links to other pages. HTML is made up of tags, which can be nested.

HTML Attributes

HTML tags can have multiple attributes. The anchor tag, for example, can be given a value that transforms a piece of text into a hyperlink like this one. Unlike anchor tags, image tags are void tags as they have no closing tags.

Whitespace

In HTML, text on multiple lines renders as a single line unless you tell it to render otherwise. Some options for doing this include the BREAK tag, which is a void tag that creates multiple lines, or the P tag, which is a block, or an invisible box around the content being rendered.

HTML Documents

An HTML document contains a number of elements, including formatting, headers (which tell the browser such details as page title via metadata), the body (where we're currently working), and others.

Computers Aren't That Bright

Computers take things ultra-literally. They can't compensate for bad programming and can fail, crash, or otherwise misbehave in the event that they're provided with bad information, unlike a human.

Lesson 2: Creating a Structured Document

About Front-end Development

Front-end development is inherently visual. It allows users to share content using HTML, CSS, and Javascript, though this course focuses only on HTML and CSS. Both HTML and CSS change over time, but the underlying principles are the same.

What Are HTML and CSS for?

If likening a webpage to a house, HTML provides the basic structure of the house: it provides the walls, the floors, etc. CSS dictates the style - the carpeting, the decorations. Javascript, in this analogy, is responsible for interactive objects: remotes, openers, and so on.

Tree-like Structures

A webpage can be thought of like a tree. There are many branches, each of which is an element, and there can be elements within elements, much like there are branches connected to branches connected to branches. One can use developer tools to see what this structure looks like as the browser interprets it in order to render the page for the viewer. While not every element is visible, all elements on the page have a rectangular box that corresponds to a line of code, and if one digs deep enough in the inner structure one can find every line of text presented on the page.

How Browsers Know What To Do

HTML has a specific syntax and rules one must follow when using it. The DOM (document object model) determines how HTML elements are rendered, and CSS allows the user to take specific syntax and rules to change how elements such as font size, text color, etc look on a page. There are specific rules for everything, and once one can parse these elements it becomes easy to determine how a page looks from the code and how to break a page down into chunks of manageable code.

All Objects Are Boxes

In Object-oriented Web design, every element on the page has a corresponding box. This kind of box can be defined with a DIV tag, and the best way to mark styles is with Class Attributes, which serve like labels on a box. Tags can also have multiple classes.

Lesson 3: CSS

Don't Repeat Yourself!

In programming, it's inefficient to write the same code over and over again. CSS - Cascading Style Sheets - allow one to write code once and apply it to many different elements on a page. Several CSS files can be referenced in an HTML document, and all of them will be used to determine the final and visible style for the page.

An Analog: CSS is Like...

In CSS, the most specific rule applies to every element on a page. If it were a series of rules, it would be like one's personal house rules carried the same weight as a national law.

Inheritance

CSS functions on an ancestor-descendant relationship. Properties apply not only to specific elements but to all descendants. It relies on a document tree hierarchy of (X)HTML elements based on nests. Descendant elements may inherit CSS values from ancestor elements, and descendant elements inherit text properties, but box properties aren't inherited.

Inheritable and Uninheritable Properties

Text styling, whitespace, and wordspace are all inheritable.
Background, border, display, float/clear, height/width, margin, and min/max are not.

Three Ways to Write CSS

There are three ways to write CSS:
1. In the head of an HTML file (good for small projects)
2. By linking HTML to CSS (good for larger ones)
3. By writing style inline with HTML (which leads to lots of repeated code)

Boxes Revisited

The box model consists of Margins, Borders, Padding, and Content. Using CSS, one can use box-sizing, as well as border-box to provide the element and the padding. When one sets the width of the overall box, the size of the box always stays the same. Margins are NOT included in this size. This can also be set as a percentage, though this can lead to a bad UX.

Positioning Elements and Flexbox Layouts

Positioning elements is one of the most complicated things in CSS. Flexbox is an efficient way to lay out lines and DIVs. To place elements next to each other, change the value of the parent container to FLEX.

Code, Test, Refine

Frontend development requires steps.
1. Get the structure right - visibly boxify the page.
2. Work on tags.
3. Establish how big everything should be.
4. Figure out where everything needs to be positioned.
5. Work on smaller details - look for natural boxes, repeated styles, write your HTML, apply styles from the biggest/most sidewide to the smallest/most page-specific, and then fix things.
